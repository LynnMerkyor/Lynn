Lynn CLI / Lynn Code

给开发者和其他智能体的安装方式

Lynn CLI 当前最新为 v0.85.7。它是 Lynn 的终端版:跑在命令行里的 AI 编码助手,带终端 TUI、Markdown 渲染、流式输出、工具调用和长任务续跑。 一行命令装好,不用克隆仓库、不用编译;也可以被 Claude Code、Codex CLI、Kimi Code、Qwen CLI、CI 或 Lynn GUI Fleet 以无交互方式调用。

Node.js 20 LTS 或 22 LTS

1. 先准备 Node 环境

如果机器已经有 Node 20+ 和 npm，可以直接跳到下一步。先用 node -v 确认版本不低于 v20。

node -v

# macOS(Homebrew)
brew install node@20

# macOS / Linux(nvm)
nvm install 20 && nvm use 20

# Windows
winget install OpenJS.NodeJS.LTS
首次安装 / 覆盖升级

2. 安装 Lynn CLI

第一次安装和覆盖旧版本都用这条命令。--force 首次安装也无害,并能覆盖旧的 lynn/Lynn 命令软链接,避免 npm 提示 EEXIST。包走腾讯云镜像,国内直连快。

npm install -g --force https://download.merkyorlynn.com/downloads/cli/lynn-cli-0.85.7.tgz
启动命令

3. 启动 Lynn

默认走 Brain V2 路由:本地 Lynn Brain 可用时优先本地,不可用时自动回到远端 Brain。v0.85 的普通 GUI/CLI 对话、Lynn -p 和编码执行统一走自研 core + StepFun 3.7 Flash 主链:先拿证据,再快速总结,必要时才 fallback。实验性 manager 仍可通过 Lynn manager run -p ... --jsonl 显式调用,不会抢占默认路径。

Lynn                 # 交互式聊天 TUI
Lynn code            # 编码 agent TUI
Lynn manager run -p "拆分并验收这个长任务" --jsonl
Lynn --version       # 应输出 0.85.7
Lynn agents          # 输出可复制的 Fleet / 其他 CLI 调用方式

# 无交互编码 worker 示例
Lynn code -p "fix tests, run the suite, summarize the diff" \
  --json \
  --cwd /path/to/worktree \
  --approval yolo \
  --sandbox danger-full-access \
  --save-session

安装前你可能想知道

我怎么知道我的 Mac 是 Apple Silicon 还是 Intel？

点击左上角苹果菜单，选择“关于本机”。如果看到 M 系列芯片，就是 Apple Silicon；如果看到 Intel，就是 Intel 版。

Windows 弹出安全提示还能装吗？

可以。正式发布安装包会完成 Windows 代码签名；SmartScreen 仍可能因为新版应用声誉积累不足而提示确认。

我想看更新说明或校验原始发布记录怎么办？

你可以直接打开 Gitee Release 页面查看所有历史版本、更新说明和备用下载入口。

V0.85.7:27B Coding Q4 MTP 默认端侧模型

V0.85.7 将本地端侧默认推荐切到公开可下载的 Qwen3.6-27B Coding GGUF Q4 imatrix MTP。 低配设备不主动弹安装引导,仍可在模型页手动选择 9B / 4B 降级；32GB+ 用户仍可显式选择 35B-A3B Q5_K_M。

  • 明确读取 `/Users/.../main.tex` 这类绝对路径时,按目标文件读取,不回退 workspace 目录
  • 解释 `file://` 协议问题时不再把它当成本地路径预取
  • 上一轮 ComfyUI、main.tex 或其它文件任务不会污染下一轮普通追问
  • 大文件只做可控 preview,避免巨大 LaTeX/代码文件塞满上下文
  • Windows `D:\...`、`D:/...` 和 `%20` 编码路径兼容性补强
  • Windows 安装包热修:后台工具、MCP、本地模型、语音和 worker 子进程不再反复弹 CMD 窗口
  • 默认下载公开 27B Coding Q4 四分片 GGUF,校验后由 llama.cpp 以 MTP 启动
  • 保留上版会话进展、低配不主动弹本地模型引导和 9B/4B 降级入口
  • Agent regression、CLI200、GUI100 和 release preflight 继续作为发版门禁
