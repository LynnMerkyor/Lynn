Lynn CLI / Lynn Code

给开发者和其他智能体的安装方式

Lynn CLI 当前最新为 v0.80.9。它是 Lynn 的终端版:跑在命令行里的 AI 编码助手,带终端 TUI、Markdown 渲染、流式输出、工具调用和长任务续跑。 一行命令装好,不用克隆仓库、不用编译;也可以被 Claude Code、Codex CLI、Kimi Code、Qwen CLI、CI 或 Lynn GUI Fleet 以无交互方式调用。

Node.js 20 LTS 或 22 LTS

1. 先准备 Node 环境

如果机器已经有 Node 20+ 和 npm，可以直接跳到下一步。先用 node -v 确认版本不低于 v20。

node -v

# macOS(Homebrew)
brew install node@20

# macOS / Linux(nvm)
nvm install 20 && nvm use 20

# Windows
winget install OpenJS.NodeJS.LTS
首次安装 / 覆盖升级

2. 安装 Lynn CLI

第一次安装和覆盖旧版本都用这条命令。--force 首次安装也无害,并能覆盖旧的 lynn/Lynn 命令软链接,避免 npm 提示 EEXIST。包走腾讯云镜像,国内直连快。

npm install -g --force https://download.merkyorlynn.com/downloads/cli/lynn-cli-0.80.9.tgz
启动命令

3. 启动 Lynn

默认走 Brain V2 路由:本地 Lynn Brain 可用时优先本地,不可用时自动回到远端 Brain。模型级联是 StepFun 3.7 Flash(256K 上下文,high 推理,32K 推理/生成预算) → MiMo → Spark。如果只想给其他智能体或脚本调用,请使用 Lynn code -p ... --json。

Lynn                 # 交互式聊天 TUI
Lynn code            # 编码 agent TUI
Lynn --version       # 应输出 0.80.9
Lynn agents          # 输出可复制的 Fleet / 其他 CLI 调用方式

# 无交互编码 worker 示例
Lynn code -p "fix tests, run the suite, summarize the diff" \
  --json \
  --cwd /path/to/worktree \
  --approval yolo \
  --sandbox danger-full-access \
  --save-session

安装前你可能想知道

我怎么知道我的 Mac 是 Apple Silicon 还是 Intel？

点击左上角苹果菜单，选择“关于本机”。如果看到 M 系列芯片，就是 Apple Silicon；如果看到 Intel，就是 Intel 版。

Windows 弹出安全提示还能装吗？

可以。正式发布安装包会完成 Windows 代码签名；SmartScreen 仍可能因为新版应用声誉积累不足而提示确认。

我想看更新说明或校验原始发布记录怎么办？

你可以直接打开 GitHub Release 页面查看所有历史版本、更新说明和备用下载入口。

本次版本重点

  • 前置缓存命中可见:usage、session、replay 和 Lynn cache doctor --json 统一显示 prefix-cache ... hit,不增加 ctx% 焦虑条
  • 长任务运行时压缩:旧轮次自动压缩,保留原始目标、当前计划和最近工具结果,JSONL 发出 code.runtime.compacted
  • Brain 早期断流自动重试:SSE 在可见输出前断开会退避重试,输出开始后不重试以避免重复半轮工具调用
  • Brain 默认路由为 StepFun 3.7 Flash(256K 上下文,high 推理,32K 推理/生成预算) → MiMo → Spark Qwen 3.6 35B A3B
  • CLI 镜像安装、headless agent contract、longrun 压缩 smoke 和 Fleet 门禁进入发布流程
