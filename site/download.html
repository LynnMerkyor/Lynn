Lynn CLI / Lynn Code

给开发者和其他智能体的安装方式

Lynn CLI 是 V0.80 的终端入口。它可以由人手动使用，也可以被 Claude Code、Codex CLI、Kimi Code、Qwen CLI、CI 或 Lynn GUI Fleet 以无交互方式调用。

Node.js 20 LTS 或 22 LTS

1. 先准备 Node 环境

如果机器已经有 Node 20+ 和 npm，可以直接跳到下一步。

# macOS(Homebrew)
brew install node@20

# macOS / Linux(nvm)
nvm install 20 && nvm use 20

# Windows
winget install OpenJS.NodeJS.LTS
首次安装 / 覆盖升级

2. 安装 Lynn CLI

第一次安装和覆盖旧版本都用这条命令。`--force` 可以覆盖旧的 `lynn/Lynn` 命令软链接,避免 npm 提示 EEXIST。

npm install -g --force https://download.merkyorlynn.com/downloads/cli/lynn-cli-0.80.0.tgz
启动命令

3. 启动 Lynn

默认走本地 Lynn Brain 路由；如果只想给其他智能体或脚本调用，请使用 `Lynn code -p ... --json`。

Lynn                 # 交互式聊天 TUI
Lynn code            # 编码 agent TUI
Lynn agents          # 输出可复制的 Fleet / 其他 CLI 调用方式

# 无交互编码 worker 示例
Lynn code -p "fix tests, run the suite, summarize the diff" \
  --json \
  --cwd /path/to/worktree \
  --approval yolo \
  --sandbox workspace-write \
  --save-session

安装前你可能想知道

我怎么知道我的 Mac 是 Apple Silicon 还是 Intel？

点击左上角苹果菜单，选择“关于本机”。如果看到 M 系列芯片，就是 Apple Silicon；如果看到 Intel，就是 Intel 版。

Windows 弹出安全提示还能装吗？

可以。正式发布安装包会完成 Windows 代码签名；SmartScreen 仍可能因为新版应用声誉积累不足而提示确认。

我想看更新说明或校验原始发布记录怎么办？

你可以直接打开 GitHub Release 页面查看所有历史版本、更新说明和备用下载入口。

本次版本重点

  • Lynn CLI / Lynn Code 新增 Ink TUI、JSONL headless 模式和 Fleet worker adapter
  • GUI Fleet 支持多 worker 派单、日志、diff、gate、合并与远端 push 闭环
  • Brain 默认路由为 StepFun 3.7 Flash high+32K → MiMo → Spark Qwen 3.6 35B A3B
  • 链式工具、搜索、tool-storm、长任务续跑和 cache-friendly context layers 加固
  • 本地 Qwen3.5-9B MTP 改为显式启用,不会随启动自动占用约 6GB 显存/统一内存
  • CLI 镜像安装、headless agent contract 和 packaged CLI 门禁进入发布流程
