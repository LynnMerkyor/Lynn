Lynn CLI / Lynn Code

给开发者和其他智能体的安装方式

Lynn CLI 当前最新为 v0.85.5。它是 Lynn 的终端版:跑在命令行里的 AI 编码助手,带终端 TUI、Markdown 渲染、流式输出、工具调用和长任务续跑。 一行命令装好,不用克隆仓库、不用编译;也可以被 Claude Code、Codex CLI、Kimi Code、Qwen CLI、CI 或 Lynn GUI Fleet 以无交互方式调用。

Node.js 20 LTS 或 22 LTS

1. 先准备 Node 环境

如果机器已经有 Node 20+ 和 npm，可以直接跳到下一步。先用 node -v 确认版本不低于 v20。

node -v

# macOS(Homebrew)
brew install node@20

# macOS / Linux(nvm)
nvm install 20 && nvm use 20

# Windows
winget install OpenJS.NodeJS.LTS
首次安装 / 覆盖升级

2. 安装 Lynn CLI

第一次安装和覆盖旧版本都用这条命令。--force 首次安装也无害,并能覆盖旧的 lynn/Lynn 命令软链接,避免 npm 提示 EEXIST。包走腾讯云镜像,国内直连快。

npm install -g --force https://download.merkyorlynn.com/downloads/cli/lynn-cli-0.85.5.tgz
启动命令

3. 启动 Lynn

默认走 Brain V2 路由:本地 Lynn Brain 可用时优先本地,不可用时自动回到远端 Brain。v0.85 的普通 GUI/CLI 对话、Lynn -p 和编码执行统一走自研 core + StepFun 3.7 Flash 主链:先拿证据,再快速总结,必要时才 fallback。实验性 manager 仍可通过 Lynn manager run -p ... --jsonl 显式调用,不会抢占默认路径。

Lynn                 # 交互式聊天 TUI
Lynn code            # 编码 agent TUI
Lynn manager run -p "拆分并验收这个长任务" --jsonl
Lynn --version       # 应输出 0.85.5
Lynn agents          # 输出可复制的 Fleet / 其他 CLI 调用方式

# 无交互编码 worker 示例
Lynn code -p "fix tests, run the suite, summarize the diff" \
  --json \
  --cwd /path/to/worktree \
  --approval yolo \
  --sandbox danger-full-access \
  --save-session

安装前你可能想知道

我怎么知道我的 Mac 是 Apple Silicon 还是 Intel？

点击左上角苹果菜单，选择“关于本机”。如果看到 M 系列芯片，就是 Apple Silicon；如果看到 Intel，就是 Intel 版。

Windows 弹出安全提示还能装吗？

可以。正式发布安装包会完成 Windows 代码签名；SmartScreen 仍可能因为新版应用声誉积累不足而提示确认。

我想看更新说明或校验原始发布记录怎么办？

你可以直接打开 Gitee Release 页面查看所有历史版本、更新说明和备用下载入口。

V0.85.5:会话进展与本地模型推荐

V0.85.5 把右侧栏从内部调试感较重的工作地图收敛为会话进展:当前会话、需要处理、 最近会话和打开入口更清楚。端侧模型推荐链路也同步更新:27B Q5_K_M imatrix MTP 作为默认本地推荐, 32GB+ 机器可选 35B-A3B Q5_K_M,9B / 4B 保留为低配置显式降级。

  • 右侧栏改为“会话进展”:减少地图、资料、巡检、推进中等内部术语
  • 当前会话可以直接继续输入,历史会话卡片提供明确“打开”入口
  • 没有相关会话时不再展示大块空白面板,更早会话引导到左侧搜索
  • 低配置机器不会主动弹 27B 安装引导,避免用户被端侧模型推荐打扰
  • 聊天输入区的本地模型提示可以直接准备并启动 27B 默认路线
  • 保留 V0.85.4 “重新回答”污染修复,新问题不会继承旧 prompt 或旧分支目标
  • GUI / CLI / agent-regression 门禁继续覆盖桌面、终端、Brain V2、runtime frames 和发布静态校验
