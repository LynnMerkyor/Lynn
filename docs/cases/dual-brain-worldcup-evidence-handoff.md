# 双脑案例: 世界杯证据交接从 60s 降到 12s

## 一句话结论

这次世界杯赛程/赛果查询是 Lynn 双脑工作流第一次很清晰的正样本:

**DS V4 Flash 负责理解问题、决定查什么、拿工具证据；Step 3.7 Flash 负责把已经拿到的观察结果快速整理成用户可读答案；最后 DS V4 Flash 再复核/纠错。**

在同类世界杯实时查询里,旧路径会在搜索/抓取/重试之间打转,耗时接近 **60 秒** 甚至空答;新路径在证据足够后交给 Step 3.7 Flash 压缩总结,端到端约 **12 秒** 出可用答案。

## ReAct 映射

这不是传统单模型 ReAct,而是把 ReAct 的几个职责拆给不同模型:

| ReAct 阶段 | Lynn 双脑里的角色 | 说明 |
| --- | --- | --- |
| Reason / Plan | DS V4 Flash | 理解用户问题,判断需要哪些工具和证据 |
| Act | DS V4 Flash | 驱动搜索、比分、行情、天气等工具调用 |
| Observation | 工具返回结果 | 搜索结果、结构化数据、网页抓取、行情/天气等 |
| Observation-to-Answer | Step 3.7 Flash | 不是观察本身,而是把观察结果压缩成答案 |
| Critic | DS V4 Flash | 复核是否有编造、日期错位、证据不足或格式问题 |

核心流:

```text
用户问题
  -> DS V4 Flash: 理解、拆解、调用工具
  -> 工具证据账本
  -> Step 3.7 Flash: 从证据直接整理答案
  -> DS V4 Flash: 复核 / 阻断 / 放行
  -> 用户可见回答
```

## 为什么有效

旧失败模式不是"没有搜索",而是**拿到证据后还继续搜**:

- DS V4 Flash 已经拿到了足够的搜索/赛程证据;
- runtime 继续让同一个模型追加搜索、抓页面、换 query;
- 搜索结果里混入过期新闻、百度/页面摘要、JS 空页或不相关来源;
- 模型迟迟不总结,最后出现慢答、空答、工具状态汇报或搜索结果堆砌。

新路径的关键是把任务从"继续查"切换为"用已有证据回答":

- 证据预算达到阈值后,停止无意义的追加搜索;
- 把工具结果整理成 evidence ledger;
- 交给 Step 3.7 Flash 做快速 synthesis;
- 再由 DS V4 Flash 做 critic。

Step 3.7 Flash 的价值不是更会查,而是**很快地把 Observation 压成 Answer**。这正好补上 V4 Flash 在长工具链后容易继续思考/继续行动、不愿收束的弱点。

## 触发规则

不要用"世界杯"这种关键词补丁。这个案例应该沉淀成通用 runtime 规则:

- 已有 grounded evidence 达到预算;
- 同类工具重复调用但没有新增事实;
- 工具成功但模型没有最终正文;
- 用户问的是赛程、赛果、行情、天气、表格、摘要等证据已足够回答的问题;
- 主模型超时、空正文、只输出工具状态或搜索结果堆砌;
- 答案需要事实交叉验证,但当前轮不应该继续扩大搜索面。

触发后:

1. 冻结当前证据账本。
2. 交给 Step 3.7 Flash 做 evidence-only synthesis。
3. 禁止 Step 继续自由搜索,除非明确授权。
4. DS V4 Flash 对 Step 的结论做复核。
5. 复核阻断时,优先用同一证据账本修正,不要无脑重搜。

## 设计约束

- **DS V4 Flash 是主脑**:保留理解、规划、行动、复核能力。
- **Step 3.7 Flash 是高速执行/总结脑**:尤其适合工具后总结、表格整理、行情/赛程/天气答案压缩。
- **MiMo 是后续 fallback**:可做重任务或补位,但不放在第一复核位。
- **GLM 5.0 Turbo 是最后 fallback**:并发低、易 429,不应被放在前排。
- **不要让一个模型在同一个坑里重复重试**:空答/超时/工具链无总结时立刻切换职责。
- **不要对有副作用的任务双路执行**:文件修改、删除、支付、系统命令等不能让两个模型并行乱跑。

## 成功指标

这个案例以后应该进入 CLI/GUI 门禁:

- 同类实时赛程/赛果问题不超过少量工具轮;
- provider chain 能体现 DS V4 Flash -> Step 3.7 Flash -> DS V4 Flash review;
- 答案不是搜索结果堆砌;
- 没有空正文;
- 没有 stale date / publish date 当 event date;
- 延迟接近 12 秒路径,不能回到 60 秒工具风暴路径。

## 可复用经验

这次优化说明:双脑不是为了堆模型,而是为了拆职责。

如果一个模型同时负责规划、行动、观察整理、最终验收,它会在复杂工具链里反复犹豫。把"观察到答案"这一步交给高速模型,再让主脑复核,反而更稳定、更快,也更容易解释给用户看。
